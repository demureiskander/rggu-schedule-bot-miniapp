# Changelog

Все заметные изменения проекта. Формат — [Keep a Changelog](https://keepachangelog.com/ru/),

## [Unreleased]

### Added
- Кнопка «Сегодня» в шапке расписания (прыжок на текущий день); сегодняшний день
  в полоске недели — акцентным жёлтым.
- Номер пары в карточке («1-я пара · 09:00 — 10:20»).
- Полоска недели показывает все 7 дней (Пн–Вс); дни без пар приглушаются.
  Тумблер «Подсвечивать дни без пар» в настройках (по умолчанию вкл).
- Блок «по этому предмету» в деталях пары: следующая пара, сколько осталось до
  конца семестра (с разбивкой лек/сем) и дата экзамена — из загруженного
  расписания (intervalMode=4 отдаёт весь семестр). Экзамен в «осталось» не считаем.
  Разбивка человеческая со склонением: «Осталось 4 пары: 3 лекции, 1 семинар»;
  при нуле — явно «лекций нет»/«семинаров нет»; спец/прочее — отдельной строкой.

### Fixed
- Подсчёт «по предмету»: тип «ЛЕК; СЕМ» (комбинированная пара) определялся через
  точное сравнение и попадал в «прочее». Теперь матчинг через `includes` (без
  учёта регистра); комбинированные пары — отдельная категория «N лекций-семинаров».
- Институты `ИЖиМ`, `ИЕиВИ` добавлены в словарь расшифровок.
- Шаг «Форма обучения» сгруппирован по уровню (Бакалавриат / Магистратура /
  Второе высшее) с подписями по форме посещения — вместо плоского списка из 12.
- Видимая кнопка «назад» (←) в шапке каждого шага выбора (помимо системного
  BackButton); показывается только когда есть куда вернуться.
- У расшифрованных институтов в списке — аббревиатура в скобках («… (ИЛ)»).
- Выбор группы деревом: форма → курс → **институт → направление → группа**.
  Словарь `INSTITUTES` (аббрев → полное имя) + `buildTree(flows)` в `constants.js`;
  институт из кода группы, направление/профиль из `details`. Институты сортируются
  по алфавиту названия, нерасшифрованные аббревиатуры — фолбэком как есть, в конце.
  Дубли по значению (ФРиСО/ФРИСО) схлопываются в одну ветку. Профиль — подписью
  у группы. Группы грузятся один раз на форму+курс и кэшируются на сессию.

### Fixed
- Тема на старте: единый источник истины — `store.theme` (дефолт `'dark'`).
  Убрана инициализация от `colorScheme()` Telegram и авто-следование, из-за
  которых грузилась светлая тема при тёмном тумблере (CSS и состояние расходились).

### Changed
- Маскот: финальные webp-арты из `uploads/` (wave/sleep/sad/think) вместо
  промежуточных svg.
- Экран выбора формы: убран технический код формы (1-Б-О и т.п.) из UI —
  остаётся только читаемая подпись; «Б» теперь «Бакалавриат» (вместо «Бак / спец»).

## [0.1.0] — 2026-06-14

Первый MVP: welcome → выбор группы → расписание (3 вида) + бэкенд-прокси.

### Added
- `CLAUDE.md` — контекст проекта для будущих сессий (стек, структура, контракт
  API, дизайн-токены, ключевые решения, UX-философия).
- `CHANGELOG.md` — этот файл.
- Каркас проекта (часть 1/3): структура папок, плейсхолдеры маскота, `README`,
  `.gitignore`, `manifest.webmanifest`, `favicon.svg`.
- `config.js` — `API_BASE` (плейсхолдер домена бота), версия, имя бота.
- `src/constants.js` — тайм-слоты, типы пар, формы обучения + разбор кода формы
  в человекочитаемый вид, дни недели/месяцы, иконки погоды, пути маскота.
- `src/telegram.js` — обёртка Telegram WebApp SDK: ready/expand, тема, BackButton,
  HapticFeedback, CloudStorage с безопасной деградацией вне Telegram.
- `src/store.js` — состояние + persistence (CloudStorage → localStorage),
  сессионный кэш расписания/погоды.
- `src/api.js` — запросы к `/flows`, `/schedule`, `/weather`; нормализация
  расписания в карту по датам с тайм-слотами; чистка `"-"` → пусто.
- `index.html` + `styles.css` — каркас, дизайн-токены, тёмная/светлая темы.
- `server/` — standalone Node-прокси (отдельный Railway-сервис): `/api/flows`,
  `/api/schedule`, `/api/weather` (open-meteo, кэш на сутки), CORS + preflight,
  health-check. Чистый `node:http`, без зависимостей; `package.json`, `.env.example`.

- Экраны (часть 2/3): `src/main.js` — точка входа, инициализация SDK/темы/persistence,
  стек-роутер экранов с системным BackButton; на первом запуске тема уважает
  `colorScheme` Telegram. `src/screens.js` — приветствие (6.1) и выбор группы
  форма→курс→поиск (6.2) с загрузкой/пустым/ошибкой через общий блок маскота;
  каждый шаг picker'а — кадр стека (BackButton листает шаги назад). Picker шлёт
  числовой id формы; группа сохраняется в стор и ведёт на расписание.
- Стили picker'а (option-row, course-grid, search-input) в `styles.css`.
- `src/store.js`: `isPersisted(key)` — отличать сохранённое от дефолта (для темы).
- `src/telegram.js`: BackButton переведён на `onBackButton`/`setBackVisible`
  (хендлер регистрируется один раз, видимость — отдельно).

- Расписание (часть 3/3): `src/render.js` — `renderLesson(lesson, layout)` для
  трёх видов (блочный/компактный/ленточный), бейджи типов, счётчик «осталось пар»
  (динамика для сегодня, тикер раз в минуту), бейдж погоды, полоска недели,
  навигация дня, контент деталей пары. `src/screens.js` — экран расписания: общая
  шапка, свайп/стрелки листают дни без новых запросов, пустой день (маскот спит),
  загрузка/ошибка с маскотом. Bottom sheets: детали пары и настройки (вид, погода,
  тема, смена группы) — общий паттерн с перехватом BackButton и свайпом вниз.
- `src/theme.js` — вынесены `applyTheme`/`resolveInitialTheme`; убран `window.__applyTheme`.
- Старт: нейтральный сплэш до загрузки persistence (без моргания через welcome);
  валидация сохранённой группы (form/id/name) перед переходом на расписание.
- Кэш расписания по группе с инвалидацией после 22:30 МСК (`getFreshSchedule`).
- Реальные SVG-спрайты маскота в `public/mascot/` вместо плейсхолдеров.
- Полоска недели гасит дни вне загруженного диапазона (неактивны, без тапа) —
  чтобы «пустой день» (каникулы) не путался с «вне диапазона».

### Changed
- **Финальная палитра** (Hi-Fi эталон): тёмная #0E0B18/#17132A/#201A38, акцент
  #7C6CF7, CTA/бейдж лекции #F59E0B; светлая #F6F4FB/#FFFFFF, акцент #5A43E0.
  Токены в `styles.css` заменены — компоненты перекрашены через `var()`.
- Архитектура бэкенда: вместо эндпоинтов в боте — standalone прокси в `server/`,
  независимый от бота (по обновлённому ТЗ). Погода — из open-meteo без ключа.
- Доки (CLAUDE.md, README, config.js) приведены к новой архитектуре; зафиксирован
  upstream-контракт API РГГУ и расхождение `eduform` = числовой id формы, не код.
